APT Hash Sum mismatch error

Created2020-05-07|Updated2020-05-07

VirtualBox 上のVM でsudo apt updateやsudo apt upgrade をした際にHash Sum mismatch エラーが出力されました。MD5 は一致していますが、SHA256、SHA-1 が一致しない場合があるようです。
簡単に起こるとは考えにくいですが再現性があります。これは何に起因した事象なのでしょう。
手動でダウンロードしたファイルのハッシュ値は期待していたものでした。

2020-05-10 追記


以下を有効にしていると、そもそも起動用ISO のチェック時にエラーが発生していました。

  • Linux 用 Windows サブシステム (Windows Subsystem for Linux)
  • 仮想マシンプラットフォーム (Virtual Machine Platform)

上記を無効にするとエラーは無くなり、Hash Sum mismatch も解決しました。
VM 初期構築時に上記を一時的に無効にすることがワークアラウンドのようです。

2020-05-28 追記

https://github.com/ktkr3d/ktkr3d.github.io/issues/1
解決策を教えていただきました。ありがとうございます！

apt updateもapt upgradeも正常に実行できました。
この事象はgcrypt20 モジュールに起因していたようです。

ソフトウェアバージョン

  • VirtualBox 6.1.6

  • Hyper-V 10.0.19619.1000
  • WSL2 (Ubuntu 18.04) 導入済

気づき事項

  • 発生環境について
    • VirtualBox のVM で発生した
    • Hyper-V のVM や物理環境では発生しなかった
    • apt のバージョンに依存しないようです
  • 不一致箇所について
    • SHA256 だけ異なる場合があります(MD5 とSHA-1 は一致)
    • SHA256 とSHA-1 が異なる場合があります(MD5 は一致)
    • ファイルサイズは一致
    • sudo apt -d install <package name>で不一致になるのに、wget http/***/***.deb のファイルは期待されるハッシュ値になる
  • ワークアラウンド
    • sudo apt clean しても改善せず
    • sudo rm -rf /var/cache/apt/archives/* しても改善せず
    • deb ファイルをwget でダウンロードしたものは期待したハッシュ値になる
    • deb ファイルを/var/cache/apt/archives/ に格納すると正常にアップグレードできる場合がある
