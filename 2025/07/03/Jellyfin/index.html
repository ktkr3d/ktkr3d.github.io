Jellyfin

Created2025-07-03|Updated2025-07-05

メディアサーバJellyfinを使ってみました。今まで散らばっていた大量のメディアファイルを整理することができました。

Jellyfin
The Free Software Media System

環境

  • WSL (Ubuntu 24.04.2 LTS)
    • Jellyfin 10.10.7

構築

サーバ

  • Jellyfin インストール
  • 別のマシンからアクセスできるようにWSLへのポートフォワード設定一覧に8096/tcpを追加する

クライアント

https://github.com/awesome-jellyfin/awesome-jellyfin/blob/main/CLIENTS.md
「Jellyfin Media Player」を入れておけば問題なさそうです。

  • Windows
    https://github.com/jellyfin/jellyfin-media-player/releases
  • Arch Linux
  • Android
    https://jellyfin.org/downloads/clients?platform=Android,Android%20TV

メディアファイルの配置

既存のメディアファイルが複数のフォルダに分散しています。
元のファイルはそのままにして、Jellyfinで管理しやすいフォルダ構造上にシンボリックリンクを配置することにしました。

メディアフォルダの作成

  • メディアのカテゴリのルートを作成

    /media/links は整理前の映画や番組のシンボリックリンクを仮配置するフォルダ

  • Jellyfinのメディアに割り当て
    フォルダ メディアの種類
    /media/Movies 映画
    /media/Shows 番組
    /media/Music 音楽

構造

  • 各カテゴリ毎に以下の構造にします

メディアファイル実体

  • 動画ファイルの実体フォルダ群（WindowsのFドライブ）
    /mnt/f/home/chinachu/
    /mnt/f/home/epgstation/
    /mnt/f/home/epgstation.old/
  • 音楽フォルダの実体フォルダ（WindowsのEドライブ）
    /mnt/e/home/Music/

シンボリックリンクの作成

  • 音楽
    /media/Music/に配置します。
    AIにbashスクリプトを作ってもらいます。
    「ファイルの ARTIST、ALBUM、TITLE タグを使って、ファイルをディレクトリ階層 “ARTIST/ALBUM/TITLE.拡張子” に移動する Bash スクリプトを作成」のような感じで指示すると素晴らしい精度でスクリプトを生成してくれます。
  • 映画と番組
    • 映画の場合
      ファイル名を「タイトル_(YYYY).xxx」の形式に変更する。
    • 番組の場合
      ファイル名を「タイトル_S01E01.xxx」の形式に変更する。

その他

  • プラグイン、テーマ等
    https://github.com/awesome-jellyfin/awesome-jellyfin/
  • Jellyfinのメディアライブラリの種類
    「映画」、「音楽」、「番組」、「ブック」、「ホームビデオと写真」、「ミュージックビデオ」、「TV番組と映画を混ぜる」
  • Jellyfinはシンボリックリンクを認識する
  • 複数の機器のクライアントで同時にメディアを再生するには：
    別のアカウントを利用すべきかもしれない
